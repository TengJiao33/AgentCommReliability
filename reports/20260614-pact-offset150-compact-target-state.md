# PACT Offset150 Compact Target-State GPU Run

## What We Tried

I tested whether target preservation could be made cheaper and more structured
than the verbose target-slot contract.

The new prompt flag was:

```text
PACT_COMPACT_TARGET_STATE=1
```

It adds a short public field:

```text
Target Slot: [answer type; anchor entity or entities; required qualifier]
```

Run record:

- `experiments/20260614-1642-a8002-pact-qwen25-14b-hotpot50-offset150-compact-target/`

## What Happened

| Arm | EM | Avg F1 | Avg final-answer words | Avg comm tokens | Avg total tokens |
| --- | ---: | ---: | ---: | ---: | ---: |
| final-answer contract | `25/50` | `0.6777` | `2.02` | `324.6` | `4662.2` |
| compact target + final contract | `22/50` | `0.6036` | `2.06` | `497.3` | `5427.8` |
| compact target only | `19/50` | `0.5790` | `5.18` | `547.4` | `5471.5` |

Adding compact target state on top of the final-answer contract:

- loses `3` exact matches;
- drops average F1 by `0.0741`;
- adds `172.7` communication tokens/sample;
- creates 8 right-to-wrong cases and 5 wrong-to-right cases.

Without the final-answer contract, compact target state is worse:

- `19/50` EM;
- final answers become longer again;
- communication tokens rise to `547.4` per sample.

## Target Slot Field Audit

The field is usually emitted:

| Arm | Events with Target Slot | All-turn samples | Stable target-slot samples | Generic final targets |
| --- | ---: | ---: | ---: | ---: |
| compact target + final | `199/200` | `49/50` | `27/50` | `0` |
| compact target only | `199/200` | `49/50` | `24/50` | `0` |

This matters. The failure is not format compliance. The model mostly writes the
field. The failure is that the field is unstable across agents and turns.

## Diagnostic Cases

Sample `199` is the cleanest sentinel.

The final-answer contract alone answers correctly:

```text
1999 Odisha cyclone
```

With compact target state, the target slot alternates:

```text
[film; 1999 Odisha cyclone; 2007]
[film; Cyclone Gonu; 2007]
[film; 1999 Odisha cyclone; 2007]
[film; Cyclone Gonu; 2007]
```

The final answer becomes:

```text
Kathantara
```

That is not a pure final-answer surface failure. It is a public target-state
oscillation: the agents disagree about whether the answer target is the cyclone
or the film.

Sample `198` shows entity drift inside the target field:

```text
[number; Abdurahman Khadr; settlement in 2017]
...
[number; Omar Khadr; settlement in 2017]
```

The answer remains a high-F1 near miss, but the target-state change is visible.

## Interpretation

This is a useful negative result.

The compact target-state field did not solve target preservation. It exposed a
harder interface problem:

```text
target state needs to be derived and preserved, not repeatedly regenerated.
```

A free-form LLM-generated target slot becomes another place where agents can
negotiate, overwrite, or oscillate between plausible targets. That explains why
field compliance can be high while EM, F1, and token cost get worse.

## Consequence For The Idea

The project should not pitch "add a Target Slot field" as a method.

The stronger idea is:

```text
public-state communication needs target-state consistency checks or frozen
target projection, not just more structured text fields.
```

The next target-preservation intervention should freeze a compact target state
from the original question before the agent exchange, then require later agents
to reference it rather than regenerate it.

## Caveats

- One 50-sample slice.
- Qwen2.5-14B, not Qwen3-14B.
- The target-slot drift diagnostic is lexical.
- The compact field was generated by the same LLM in every turn, so this tests
  generated target state, not deterministic target extraction.
