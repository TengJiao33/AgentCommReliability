# 研究进展综合报告：我们目前到底摸出了什么

日期：2026-06-16

状态：基于当前本地报告、实验摘要、项目日志、证据登记表，以及最新
TypeCastArena 运行结果的阶段性综合判断。

## 一句话结论

我们现在摸到的，不是一个简单的提示词技巧，也不是一个已经成型的方法。
我们摸到的是一个还活着、而且越来越清楚的研究对象：

```text
多智能体通信中的认识论类型擦除。
```

更直白地说：

```text
问题不只是一个智能体发来的信息是真是假，而是接收方是否知道这条信息
到底是什么类型的东西：证据、推理、假设、候选答案、局部推导、记忆、
验证结果、任务契约，还是最终承诺。
```

当一个智能体的中间产物跨过通信边界，被压平成自然语言塞进另一个智能体
的上下文时，它原来的类型经常丢失。接收方就会在模型内部做一种隐式的
“错误类型转换”：把假设当事实，把候选答案当承诺，把局部推导当可继承
算子，把记忆候选当持久状态，把验证风格的文字当程序性批准。

早期我们叫它“权威生成”。现在看，“权威生成”更像是一个表层症状：
文本之所以获得下游权威，是因为它跨边界时丢了类型，接收方把它转换成了
比原本更强的东西。

目前成熟度判断：

| 问题 | 当前判断 |
| --- | --- |
| 现象是不是真的存在 | 是，足够继续追 |
| 只是“模型会受权威文本影响”吗 | 不是完全如此，数学实验里有大量非复制型算子继承 |
| 已经能写成论文了吗 | 还不行 |
| 值不值得迈大步 | 值得，但要更尖锐地加压，而不是盲目堆规模 |
| 最强的贡献形态 | 通信边界诊断和类型化协议，而不是普通提示词防御 |
| 最大弱点 | 样本选择性、案例集中、模型单一、完整多智能体闭环证据还不够 |

## 这条线是怎么长出来的

这个想法不是一开始拍脑袋定的。它是从一堆复现、失败、日志和小实验里
慢慢长出来的。

### 第一阶段：从消息保留开始

最早我们接触的是 MAD-MM、DAR、MOC 这些多智能体通信或辩论方法。
当时的问题大概是：

```text
智能体之间到底该保留、隐藏、压缩、转发哪些消息？
```

一些关键结果：

| 对象 | 结果 | 留下的东西 |
| --- | --- | --- |
| MAD-MM 的 GSM8K 小切片 | 单智能体链式推理 `0.94`，朴素多智能体辩论 `0.96`，客观遮蔽 `0.95`，主观遮蔽 `0.96` | GSM8K 太饱和，但能看到错误记忆污染 |
| MAD-MM 的基准图谱 | MATH50 上客观遮蔽最好，MMLU-Pro50 上朴素辩论略好，AIME 上没有明显收益 | 多智能体通信效果强烈依赖任务类型 |
| DAR 的 GSM8K100 | 原始关键过滤最后 `0.93`，有 3 个从对到错的转移 | 保留规则会压掉正确信息，也会保留误导信息 |
| DAR 的拆分消融 | 只保留答案能降成本并修掉一些错，保留完整消息加守卫能到 `0.96` | 答案、证据、可解析性和正确性不是一回事 |
| MOC 强制合并 | 合并路径和压缩成本被跑通，但 GSM8K 太小太饱和 | 拓扑和压缩机制能看见，但需要更合适任务 |

这一阶段排除了一个方向：

```text
这个项目不应该只是做一个更好的消息保留启发式。
```

真正留下来的，是对通信表面的敏感：同一条消息被保留为完整推理、只保留
答案、压缩摘要、公共状态字段，行为会完全不同。

### 第二阶段：PACT 把问题推到公共状态

PACT 让我们从“保留什么消息”转向“什么东西能进入公共状态”。

在 HotpotQA50 上，原始 PACT 严格匹配只有 `17/50`。加入最终答案契约后，
同样的模型、同样的通信格式，严格匹配变成 `34/50`，平均最终答案长度从
`9.92` 个词降到 `2.08` 个词。

这说明最终答案格式不是单纯的后处理问题，它会改变模型生成行为。

随后公共状态字段压力实验更关键：

| 条件 | 严格匹配 | 解读 |
| --- | ---: | --- |
| 原问题加证据，不给公共目标 | `0.590` | 去掉公共目标反而常常更好 |
| 冻结由问题导出的目标，加证据 | `0.580` | 从问题导出的目标有时有用 |
| 原问题加公共状态，不给最终答案 | `0.520` | 公共状态本身并不可靠 |
| 只给公共目标和证据，不给原问题 | `0.440` | 公共目标不能替代原问题 |
| 原问题加公共状态和最终候选 | `0.550` | 可见候选答案带来复制和承诺压力 |

这里摸出的东西是：

```text
公共状态不是天然可信、天然可复用的状态。
```

公共字段只有在目标契约、证据、最终答案承诺仍然被原问题授权时才可靠。
否则它会变成一个看起来结构化、实际上会漂移的状态对象。

### 第三阶段：候选答案可见性成为危险表面

PACT 类型化边界拆分实验非常清楚。

在正向目标聚焦样本里：

| 变体 | 结果 |
| --- | --- |
| 类型化但不给候选答案，或候选答案只作为隐藏元数据 | 新增权威违规只有 `1/26`，能救回 `20/21` 个错误任务契约失败和 `13/14` 个伪造最终答案失败 |
| 类型化但把候选答案显式展示给模型 | 新增权威违规升到 `7/26`，正向严格匹配从 `0.875` 降到 `0.625` |
| 先抽取再比较 | 稍微缓解，但远不如隐藏候选答案 |

这一步让我们意识到：

```text
即便明说“这是不可信候选答案”，只要它对模型可见，它仍可能变成最终答案锚点。
```

所以问题不是“加一个更醒目的不可信标签”就能解决。更合理的协议想法是：
候选答案有时不该出现在模型可见公共状态里，而应该作为路由器、评估器、
后验比较器的元数据存在。

### 第四阶段：权威生成

我们后来把一个重复出现的模式叫做“权威生成”：

```text
同一段文本，因为被提示成不同的未来状态，会获得不同的行为影响力。
```

PACT 权威阶梯实验支持了这个判断：

| 未来信号 | 在基线正确样本上的权威违规率 |
| --- | ---: |
| 隐藏元数据 | `0.000` |
| 原始可见提及 | `0.111` |
| 答案形状文本 | `0.167` |
| 同伴声明 | `0.333` |
| 验证器批准结果 | `0.333` |
| 保存记忆 | `0.278` |
| 当前任务要求 | `0.611` |
| 最终答案承诺 | `0.389` |

曲线不是严格单调的，这很重要。它说明模型不是在读一个抽象的“权威分数”。
但隐藏元数据干净、可见未来状态有压力，这个方向很清楚。

当时的问题是：这个发现仍然可能被批评为普通的单模型上下文权威问题。
所以需要转到数学任务和多智能体生命周期。

### 第五阶段：数学任务迁移和非复制型算子继承

MATH 权威阶梯是目前最重要的结果之一。

运行事实：

- 总行数：`670`
- 模型：Qwen2.5-14B
- 完成：`670/670`
- 基线上一轮解法：`20/20` 正确
- 隐藏元数据控制：`0/65` 权威违规
- 可见未来信号：`57/585` 权威违规
- 语义正确：`613/670`
- 语义未知：`0`

按错误产物类型拆分：

| 错误产物类型 | 违规数 | 直接错误答案采纳 | 算子候选 |
| --- | ---: | ---: | ---: |
| 错误方程表面 | 21 | 4 | 17 |
| 错误最终答案 | 18 | 10 | 8 |
| 错误数字角色绑定 | 10 | 0 | 10 |
| 错误关系骨架 | 8 | 0 | 8 |

机制审计进一步说明：

- 总共 `57` 张违规卡片。
- 只有 `14` 张是直接采纳同伴错误答案。
- `43` 张是非复制型算子候选。
- `math159_wrong_rationale` 占 `25/57`，但其中只有 `1/25` 直接采纳同伴错误答案 `7`；大多数输出 `26`，而金标和基线是 `27`。

这是非常关键的发现：

```text
同伴错误不一定以“复制最终答案”的形式伤害模型。它经常改变的是关系、
数字角色、方程表面、计数边界、任务契约或局部推导算子。
```

所以我们不能只用“有没有复制错误答案”来衡量多智能体通信失败。

### 第六阶段：从权威生成改名为认识论类型擦除

外部文献压力让我们把故事改得更准确。

太泛的说法是：

```text
大模型会被权威上下文影响。
```

这个说法不够多智能体，也不够新。提示注入、上下文污染、说服、数据和指令
混淆都已经接近这个范围。

更准确的多智能体说法是：

```text
一个智能体生成了局部计算产物，通信层把它变成同伴消息、广播、公共状态、
共享工作区、记忆、验证结果或最终候选。接收方必须判断自己允许继承
这个产物的哪一部分。
```

于是“权威生成”变成“认识论类型擦除”的一个症状。

可能被擦除的类型包括：

| 发送方原始类型 | 接收方可能发生的错误转换 |
| --- | --- |
| 证据 | 把同伴推理当成观察事实 |
| 推理 | 把推导步骤当成可复用证据 |
| 假设 | 把暂时猜测当成公共状态 |
| 局部推导 | 把局部算子当成可继承算子 |
| 候选答案 | 把候选当最终答案锚点 |
| 最终承诺 | 把一个智能体答案当系统承诺 |
| 记忆候选 | 把临时文本当持久记忆 |
| 验证器样式结果 | 把生成文字当程序性批准 |
| 任务解释 | 把同伴的目标理解当自己的答案契约 |

这是目前最好的根因框架。

### 第七阶段：类型擦除实验

第一版 MATH 类型擦除实验给了一个小但具体的信号：

- `222/222` 完成。
- 语义正确 `219/222`。
- 控制组 `0/39` 权威违规。
- 类型擦除行 `2/48` 权威违规。
- 类型保留行 `1/120` 权威违规。
- 最清楚的是 `math121`：类型擦除的共享工作区条目把 `18√3` 推到 `18√2`，但并没有复制同伴错误最终答案。

第二版加入候选答案可见性拆分后：

- `222/222` 完成。
- 语义正确 `217/222`。
- 控制组 `0/39` 违规。
- 类型擦除同伴消息 `0/24`。
- 类型擦除共享工作区条目 `2/24`，都是 `math121` 的非复制型算子漂移。
- 类型化可见候选和类型化推导臂在这个包里是干净的。
- 类型化无候选和隐藏候选臂有 `3/48` 失败，多数更像本地重解或最终行不稳定，而不是隐藏复制。

这一步保留下来的结论：

```text
共享工作区接纳可能比普通同伴消息更危险；失败可以是非复制型算子漂移。
```

同时削弱了一个简单说法：

```text
候选答案可见性不是唯一解释；类型标签本身也不是万能药。
```

但它暴露了一个新评分缺口：即使最终答案正确，模型有时仍会在不该继承的
表面上引用或处理 Agent A 的内容。所以下一步不能只看最终答案对错，还要
看边界服从性：它有没有用了本不该接纳的产物。

## 我们真正摸到的几件事

### 一，通信收益依赖任务类型

GSM8K 经常太饱和。MATH、MMLU-Pro、AIME 的结果说明，多智能体通信不是
统一提升器。任务是否需要通信、证据是否分布、是否存在冲突、是否需要多跳
证据，都会改变结论。

### 二，消息保留不是核心，表面类型才是核心之一

DAR 告诉我们，保留一条消息并不等于保留它的有用证据。完整推理、只保留
答案、短证据、公共状态字段，会让接收方做不同的继承动作。

### 三，公共状态不是自动可信的

PACT 说明，公共字段如果失去原问题和证据授权，就会成为危险压缩。结构化
字段本身不等于可靠状态。

### 四，最终答案和候选答案是特殊危险表面

最终答案契约、候选答案、伪造最终承诺都很强。尤其是候选答案即使标成
不可信，只要对模型可见，也可能成为答案锚点。

### 五，隐藏和可见的差异真实存在，但不够解释全部

隐藏元数据多次干净：

- PACT 权威阶梯：隐藏元数据 `0/18` 违规。
- MATH 权威阶梯：隐藏元数据 `0/65` 违规。

但有些无候选或隐藏候选的类型化行仍会失败，所以不只是“看不看得见”。
更深的是接收方把它转换成了什么类型。

### 六，数学任务里最强的是非复制型算子继承

这是目前最有技术含量的发现。

模型经常没有复制同伴错误最终答案，而是继承或激活了错误的：

- 关系骨架；
- 数字角色绑定；
- 方程表面；
- 计数边界；
- 目标契约；
- 局部推导步骤。

这说明“错误答案采纳率”低估了通信失败。

### 七，接纳语义很重要

同样一段文本，如果是普通同伴消息、共享工作区条目、记忆、验证结果、隔离
拒收、类型化推理、类型化候选，会产生不同行为。真正该研究的是通信生命
周期，而不是孤立文本内容。

### 八，类型化边界有用，但“类型化”不是魔法

有些类型化通道能救 PACT 的公共任务失败；有些类型化通道在 MATH 里干净；
但有些类型化隐藏候选或局部推导通道仍会保留足够的错误算子压力。

更准确的说法不是“加类型标签就好”，而是：

```text
通信协议必须规定允许的类型转换和禁止的类型转换。
```

## 已经被排除或降级的说法

### 降级一：只是做更好的消息保留

不够。失败可能发生在保留之后，也可能发生在公共状态接纳、记忆复用、
验证器样式包装之后。

### 降级二：结构化公共状态可以解决通信

不够。PACT 说明结构化字段也会漂移，也会失去原问题授权。

### 降级三：问题就是复制错误答案

不够。MATH 里多数违规是非复制型算子变化。

### 降级四：候选答案可见性就是唯一问题

不够。PACT 强烈支持候选可见性危险，但 MATH v2 说明它不能解释全部。

### 降级五：权威生成是最深机制

不够。“权威生成”现在更像类型擦除后的一类症状。

### 还不能说：我们已经有一个方法

目前不能这么说。我们有诊断、有协议雏形、有压力对象，但还没有可部署、
可比较、可泛化的方法。

## 当前故事骨架

### 旧系统在做什么

很多多智能体系统用自然语言转发消息、辩论记录、共享记忆、公共工作区、
验证器说明、最终候选列表。这些接口保留了内容，却经常压平了内容的类型。

### 我们诊断的原因

发送方产物跨边界时发生认识论类型擦除。接收方随后做错误类型转换：

- 假设变事实；
- 候选变承诺；
- 局部推导变继承算子；
- 记忆候选变持久状态；
- 同伴投票变证据；
- 验证器样式文本变程序性结果；
- 局部任务解释变全局答案契约。

### 当前贡献形态

还不是方法。最强的形态是：

```text
一个用于测量和减少错误类型转换的通信边界诊断与类型化协议框架。
```

可能发展成：

1. 错误类型转换基准或探针套件；
2. 类型化通信边界协议；
3. 能区分答案复制、算子继承、边界服从、有用证据传递的分析器；
4. 最终才可能是一个真正方法。

## 最强证据清单

1. PACT 类型化边界拆分：
   `reports/20260615-pact-typed-boundary-split-qwen25-14b.md`

   它清楚显示候选答案可见性危险，以及无候选或隐藏候选的类型化边界能救
   很多公共任务失败。

2. MATH 权威阶梯：
   `reports/20260616-math-authority-genesis-ladder-qwen25-14b.md`

   它把现象从 PACT 迁移到数学任务，并显示可见未来信号造成 `57/585`
   违规，而隐藏元数据 `0/65`。

3. MATH 机制审计：
   `reports/20260616-math-authority-genesis-mechanism-audit.md`

   它显示 `43/57` 是非复制型算子候选。

4. MATH 类型擦除第二版：
   `reports/20260616-math-epistemic-type-erasure-v2-qwen25-14b.md`

   它说明共享工作区可能比普通同伴消息更危险，同时也说明候选可见性不是
   唯一解释。

5. 发送方接收方微协议：
   `reports/20260616-math-sender-receiver-micro-protocol-qwen25-14b.md`

   它给了目前最强的通信生命周期正证据。

6. TypeCastArena live sender / receiver：
   `experiments/20260616-1427-a8002-typecast-math-live-receiver204-qwen25-14b/analysis/summary.md`

   它是负向诊断：框架跑通，但这批压力没有诱发最终答案错误。

## 最大弱点

### 一，案例集中

关键效应集中在少数数学题：

- `math121`：`18√3` 漂到 `18√2`；
- `math159`：`27` 漂到 `26`；
- `math96`：出现直接答案采纳。

这对机制发现有用，但还不能当总体比例。

### 二，包是选择性构造的

很多 packet 来自先前失败样本或人工种子标签。这适合找机制，不适合估计
自然分布频率。

### 三，模型单一

主要行为运行都在 Qwen2.5-14B、温度 0 上。换更强模型、更弱模型或不同
解码，现象可能变化。

### 四，还不是真正自然多智能体闭环

即便 sender-receiver packet 已经更接近生命周期，它仍然是受控提示实验。
TypeCastArena 更进一步，但还不是一个自然分布式证据任务里的完整多轮系统。

### 五，当前评估器漏掉“边界服从”问题

现在主要看最终答案和错误答案采纳。最新 TypeCastArena 告诉我们：即使
最终答案正确，模型也可能用了本不该接纳的 Agent A 内容。这个需要新指标。

### 六，类型化协议可能引入本地重解错误

某些类型化通道会要求模型重新检查或重新求解，这本身会带来错误。必须把
这类错误和真正的跨智能体错误类型转换分开。

## 现在应该如何定位

按照项目的故事合成标准，目前定位是：

```text
活的诊断线索和协议候选。
```

它不是死线索。它有跨任务证据、有机制词汇、有明确压力对象。

它也不是扎实论文故事。要变扎实，还需要：

- 去除案例集中；
- 人工审计违规卡片；
- 增加边界服从指标；
- 展示类型化协议能减少错误转换，同时保留有用信息；
- 至少两个任务族上的相似机制；
- 最好再加一个模型或模型规模层级。

它也还不能强行说非常新。泛化邻域里已有提示注入、数据和指令混淆、
辩论从众、上下文污染、共享状态可靠性。潜在的新点更窄：

```text
跨智能体计算产物的认识论类型安全。
```

如果我们能把“错误类型转换”操作化到现有工作没有清楚覆盖的程度，它才会
真正变得有新意。

## 如果别人问“你们到底发现了什么”

短版：

```text
我们发现，多智能体通信失败不能只用“错误消息被复制”来描述。更好的说法是：
智能体产物跨通信边界时丢失了认识论类型，接收方把候选、假设、局部推导、
记忆、验证器说明或共享工作区条目转换成了更强的状态。PACT 里表现为候选
和最终承诺压力，MATH 里表现为非复制型算子继承。
```

谨慎版：

```text
我们现在有一个活的、边界清楚的机制候选。PACT 支持候选答案和最终承诺
表面危险；MATH 支持关系、方程、数字角色产物会导致非复制型算子漂移；
发送方接收方实验说明通信生命周期状态重要。但目前证据仍然选择性强、
案例集中、模型单一，还不能说已经有可发表的方法。
```

## 暂停 GPU 时最值得做的本地工作

### 一，人工审计违规卡片

优先看：

- 发送方接收方微协议的 `20` 张违规卡；
- 类型擦除第二版的 `5` 张违规卡；
- MATH 权威阶梯的 `57` 张卡里一部分非复制型算子候选。

建议标签：

- 直接答案采纳；
- 继承算子的错误类型转换；
- 目标契约错误类型转换；
- 本地重解背景错误；
- 最终答案契约毛刺；
- 评估器语义碰撞；
- 不是真违规。

### 二，写边界服从分析器

检查：

- 在惰性或未接纳通道里是否提到 Agent A；
- 是否使用了被隔离或拒收的内容；
- 在非继承协议下是否出现“继承上一解法”语言；
- 类型声明为假设或非承诺时，模型是否仍把它当正确解法。

这会让最新 TypeCastArena 的负结果变有用，因为最终答案正确也可能违反协议。

### 三，设计去集中化来源包

先构建，不急着跑：

- 减少对 `math121`、`math159`、`math96` 的依赖；
- 增加更多方程表面、关系骨架、数字角色样本；
- 保持同一内容在惰性、同伴消息、共享工作区、记忆、类型化推理、类型化
  局部推导之间切换。

### 四，写协议规格

比再跑一次更有价值的本地工作是写一个一页规格：

- 发送方对象类型；
- 允许的类型转换；
- 禁止的类型转换；
- 接纳状态；
- 接收方义务；
- 评估标签。

这会把“类型擦除”从漂亮词变成可测协议。

## 以后如果恢复 GPU，下一步不该怎么跑

不应该直接把最新 204 行 TypeCastArena 放大。那批太容易。

更好的 GPU 压力：

1. 做去集中化、更难的 TypeCastArena。
2. 在运行前加入边界服从评分。
3. 加入有用证据臂，不只测错误产物，避免协议只是删信息。
4. 如果效应还在，再换一个模型或规模层级。

退休条件：

```text
如果同伴消息、共享状态、记忆复用这些生命周期状态，不能比自我修正、
惰性文本和普通干扰上下文解释更多行为差异，就把这个方向降级为普通
上下文权威问题。
```

晋升条件：

```text
如果错误类型转换率能被通信生命周期和类型预测，能在去集中化样本里保留，
并且类型化协议能降低错误转换同时保留有用证据，那么认识论类型擦除就可以
晋升为一个有边界的论文故事。
```

## 最后的判断

这个项目还没有产出最终方法，但已经产出了一个真正的研究对象：

```text
跨智能体认识论类型安全。
```

当前证据说明，多智能体通信不能只问“发了什么信息”和“信息对不对”。还要问：

```text
接收方被允许对发送方产物做什么？
```

错误候选、局部推导、同伴假设、共享工作区条目、记忆、验证器风格说明，
不是同一种东西。通信通道如果擦掉这种差别，接收方就可能继承错误类型的
状态。

这就是我们现在真正摸到的东西。
