# 面向接收方的互补潜信息通信

日期：2026-07-10

## 一、创新点

### 一句话

> 现有潜在通信方法传递发送方总体重要的内部状态；我们根据接收方当前已经拥有的信息，动态提取并传递对该接收方具有新增任务价值的内部状态。

### 完整陈述

现有多智能体潜在通信通常传递完整隐藏状态、完整键值缓存，或者根据发送方注意力和全局任务表现选择部分层、部分注意力头。这些方法主要判断“发送方的哪部分状态重要”，没有直接判断“这部分信息对当前接收方是否仍然有用”。

我们同时读取发送方和接收方的当前内部状态，先估计发送方状态中能够由接收方状态预测的部分，再把不可预测部分作为候选增量信息。随后，任务效用门控从这些候选信息中选择能够改善接收方后续推理的部分，并允许系统在没有正向增量信息时不通信。

因此，通信内容由三项共同决定：

```text
发送方拥有什么
接收方已经有什么
当前任务还需要什么
```

## 二、要解决的问题

### 2.1 完整状态传递混合了四类信息

发送方内部状态并不等于有效通信内容。它同时包含：

| 信息类型 | 对接收方的价值 |
| --- | --- |
| 接收方已经拥有的正确信息 | 重复 |
| 接收方尚未拥有且与任务相关的信息 | 有效增量 |
| 接收方尚未拥有但与任务无关的信息 | 新噪声 |
| 会把接收方带偏的信息 | 有害增量 |

完整隐藏状态或完整键值缓存会把四类信息一起送入接收方。

### 2.2 发送方重要不等于接收方需要

某个状态对发送方自己的答案很重要，不代表它对每个接收方都有相同价值。

同一条代数关系面对三个接收方时，可以分别是：

- 接收方甲已经掌握的重复信息；
- 接收方乙缺失的关键条件；
- 接收方丙当前错误路径下无法利用的信息。

通信选择必须随接收方状态变化。

### 2.3 向量差异不等于互补信息

直接计算发送方和接收方状态的夹角，只能说明两个向量不同。随机噪声也可能与接收方状态高度不同。

互补信息需要同时满足：

1. 接收方当前状态不能解释它；
2. 它能够改善接收方的任务结果。

所以本方法把“新增性”和“任务效用”分成两个连续步骤。

## 三、现有工作盘点

### 3.1 选择性键值缓存通信

选择性键值缓存通信根据发送方注意力重要性和层位置先验，选择部分缓存层传给接收方。该方法能够降低通信量，但其层选择在校准后基本固定，主要反映发送方状态的全局重要性。

来源：`papers/kvcomm-selective-kv-sharing-2510.03346v3.pdf`。

### 3.2 缓存到缓存通信

缓存到缓存通信学习发送方缓存到接收方缓存的投影和融合，并通过门控调节不同接收层的融合强度。它解决的是跨模型对齐和融合位置，没有显式分解接收方已有信息与发送方新增信息。

来源：`papers/cache-to-cache-direct-semantic-communication-2510.03215.pdf`。

### 3.3 潜在缓存流

潜在缓存流把发送方完整缓存池化为固定大小的摘要，并与接收方缓存联合投影，实现跨上下文传递。它使用“传递目标模型没有的新信息”作为动机，但机制没有单独估计接收方可预测部分，也没有逐项筛选发送方增量。

来源：`papers/latent-cache-flow-2605.22863.pdf`。

### 3.4 稠密潜在通信

稠密潜在通信通过随机遮挡和稀疏恢复，估计不同注意力头对任务准确率的贡献，再保留全局得分较高的键值缓存组。

它证明了潜在状态中的任务信息具有结构性，也证明了随机选择会破坏性能。其重要性排序针对任务和模型整体，不随当前接收方状态逐题变化。

来源：`papers/dense-latent-communication-heterogeneous-agents-2606.13594.pdf`。

### 3.5 思想通信

思想通信把多个智能体隐藏状态编码到共享潜在空间，分解共享思想和私有思想，再根据智能体与潜在维度的依赖结构进行个性化路由。

它已经实现潜在因素分解和个性化传递，但选择依据是潜在结构、智能体依赖和一致程度，不是某条潜在信息对当前接收方答案的边际改善。

来源：`papers/thought-communication-multiagent-collaboration-2510.20733.pdf`。

### 3.6 选择性共享记忆

选择性共享记忆训练控制器判断哪些中间步骤值得进入全局记忆，用于减少多个智能体团队的重复计算。它处理的是可读步骤、工具结果和共享记忆，不是接收方内部状态条件下的细粒度潜在成分选择。

来源：[选择性共享记忆](https://arxiv.org/abs/2602.05965)。

### 3.7 当前空位

现有潜在通信选择机制主要属于三类：

| 选择中心 | 代表做法 |
| --- | --- |
| 发送方中心 | 根据发送方注意力、层和隐藏状态选择 |
| 全局任务中心 | 根据整体验证集准确率和消融重要性选择 |
| 表示结构中心 | 根据共享因素、私有因素和潜在依赖选择 |

新的机制采用第四种选择中心：

> **接收方增量价值中心：逐题比较发送方和接收方状态，只传递接收方当前缺失且能够改善任务结果的潜在成分。**

## 四、方法结构

### 4.1 通信单位

在每个通信时刻，发送方内部状态被拆成多个候选单元。候选单元可以对应：

- 一个模型层；
- 一个注意力头或键值缓存组；
- 一个推理步骤；
- 一个局部词元片段；
- 一个压缩潜在向量。

统一记作“候选潜在成分”。方法不对完整状态做一次性全局判断，而是对每个候选成分分别计算新增性和任务效用。

### 4.2 接收方状态表示

接收方在当前通信时刻保留自己的内部状态。该状态用于表示：

- 接收方已经编码的题目信息；
- 当前推理方向；
- 当前中间变量和局部关系；
- 当前尚未解决的不确定部分。

发送方和接收方使用同一模型时，可以直接比较对应层和对应成分。模型不同时，先使用轻量映射投到共同空间。

### 4.3 可预测部分估计

对每个候选成分训练一个轻量预测器：

```text
可预测发送状态 = 预测器（接收方当前状态）
```

预测器学习发送方和接收方之间重复、共享和可相互推断的表示部分。

### 4.4 增量状态提取

发送方候选状态减去可由接收方预测的部分：

```text
增量状态 = 发送方候选状态 - 可预测发送状态
```

增量状态表达的是：

> 发送方当前拥有、但接收方当前状态尚不能解释的内部变化。

### 4.5 任务效用门控

新增信息中仍然包含无关差异和随机噪声。任务效用门控继续读取：

- 发送方候选状态；
- 接收方当前状态；
- 增量状态；
- 双方局部一致程度；
- 接收方当前不确定程度。

门控为每个候选成分输出一个权重：

| 权重状态 | 含义 |
| --- | --- |
| 接近零 | 不传递 |
| 较小正值 | 降权传递 |
| 较大正值 | 重点传递 |

所有候选成分都可以被关闭。

### 4.6 稀疏消息构造

最终潜在消息由少量正向增量成分组成：

```text
潜在消息 = 各候选增量状态 × 对应任务效用权重
```

该消息不再等于发送方完整隐藏状态或完整键值缓存，而是针对当前接收方生成的个性化增量消息。

### 4.7 接收方融合

筛选后的潜在消息可以接入现有融合接口：

- 加到接收方对应层隐藏状态；
- 拼入接收方对应层键值缓存；
- 通过交叉注意力读取；
- 作为短潜在前缀进入接收方。

本创新点位于融合之前：决定哪些发送方内部成分值得进入当前接收方。

## 五、训练方法

### 5.1 冻结主干模型

发送方和接收方主干模型保持冻结，只训练：

1. 重复信息预测器；
2. 任务效用门控；
3. 必要的空间映射；
4. 注入强度参数。

### 5.2 训练目标

训练目标包含三项：

```text
接收方答案损失
+ 通信量惩罚
+ 重复信息惩罚
```

答案损失使门控选择真正有助于任务的潜在成分；通信量惩罚使消息保持稀疏；重复信息惩罚避免重新传递接收方已有内容。

### 5.3 状态配对

训练数据按发送方和接收方的结果关系组织：

| 发送方 | 接收方 | 主要学习内容 |
| --- | --- | --- |
| 正确 | 错误 | 识别能够救助接收方的增量信息 |
| 错误 | 正确 | 抑制会污染接收方的信息 |
| 正确 | 正确 | 删除双方重复信息 |
| 错误 | 错误 | 识别局部有效步骤与共同错误 |

### 5.4 推理过程

每次通信执行以下过程：

```text
读取发送方候选状态
读取接收方当前状态
预测接收方已经拥有的部分
计算发送方增量状态
估计每个增量的任务效用
关闭无用、重复和有害成分
融合剩余潜在消息
接收方继续推理
```

## 六、与多智能体辩论的关系

发送方和接收方是一次有向通信中的角色，不是固定身份。

在多轮辩论中：

1. 每个智能体独立形成当前推理状态；
2. 每个智能体面向不同接收者构造不同潜在消息；
3. 每个接收者融合其他智能体提供的互补信息；
4. 智能体继续推理并在下一轮交换角色；
5. 最后一轮生成答案并投票或汇总。

因此，上层仍然是多智能体辩论；本方法替换的是向所有智能体广播相同消息的通信方式。

创新点可以按照辩论场景表述为：

> 现有多智能体辩论向不同智能体广播相同答案、解释或内部状态；我们根据每个接收者当前的推理状态，分别构造其尚未拥有且有助于后续推理的潜在消息。

## 七、核心实验结构

### 7.1 主比较

| 条件 | 含义 |
| --- | --- |
| 无通信 | 接收方独立推理 |
| 完整状态通信 | 传递发送方完整状态 |
| 全局重要性选择 | 传递总体重要的层或注意力头 |
| 仅增量残差 | 去除可预测部分，不使用任务效用门控 |
| 仅任务效用门控 | 不去除可预测部分，直接门控发送方状态 |
| 接收方条件化增量通信 | 完整方法 |

### 7.2 关键对照

| 对照 | 检查内容 |
| --- | --- |
| 随机同范数增量 | 排除一般激活扰动 |
| 其他题增量 | 检查题目相关性 |
| 错误发送方增量 | 检查污染抑制 |
| 相同发送方、不同接收方 | 检查选择是否真正依赖接收方 |
| 打乱接收方状态 | 检查接收方条件是否被实际使用 |

### 7.3 评价指标

| 指标 | 含义 |
| --- | --- |
| 准确率 | 最终任务表现 |
| 救助率 | 错误接收方被改正确的比例 |
| 污染率 | 正确接收方被改错误的比例 |
| 净救助率 | 救助率减去污染率 |
| 平均通信成分数 | 实际传递量 |
| 重复信息抑制率 | 接收方已有成分被关闭的比例 |
| 接收方条件敏感度 | 同一发送方面对不同接收方时选择结果的差异 |

### 7.4 决定性现象

同一发送方状态面对不同接收方时，通信选择必须不同：

- 已经掌握某条解法的接收方不再接收该方向；
- 缺少该解法的接收方接收相应增量状态；
- 已经拥有全部关键信息的接收方不通信；
- 正确接收方不被错误发送方破坏。

这一现象把本方法与固定层选择、全局注意力头排序和发送方中心门控区分开。

## 八、论文贡献表述

### 问题贡献

现有潜在通信把发送方内部状态视为统一消息，本工作把通信价值定义为相对于特定接收方当前状态的增量任务价值。

### 方法贡献

提出接收方条件化的潜在通信机制，通过可预测部分消除和任务效用门控，构造稀疏、个性化的潜在消息。

### 实验贡献

在相同发送方、不同接收方的控制设置下，直接检验通信内容是否随接收方知识状态变化，并分别报告救助、污染、通信量和接收方条件敏感度。

### 最精简表述

> 现有方法问“发送方的什么状态重要”，我们问“发送方的什么状态对这个接收方仍然重要”。
