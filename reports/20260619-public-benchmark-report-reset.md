# 对外报告口径修正：公开基准优先

日期：2026-06-19

## 核心判断

对外报告必须以公开基准评测为主。HSA-v0 是内部派生诊断包，只能说明机制是否有信号，不能单独证明方法有效。

当前最诚实的结论是：SSEAC 已经有一个清楚的方法形状和若干内部机制信号，但公开基准上的方法优势还没有成立。下一步必须回到 PerspectiveGap / PG40 这类公开基准或公开切片，在同一模型、同一预算下做可复现对比。

## 对外能说什么

可以说：我们研究的是多智能体通信中的证据准入问题。自由文本通信会把证据发现、来源授权、范围控制、预算控制和最终承诺混在一起，导致模型在不该接收证据时接收证据，也会在证据不足时硬答。

可以说：我们的方法把模型生成候选证据和确定性约束执行分开。模型负责提出候选证据单元，编译器负责执行来源、范围、验证状态、预算和充分性约束。

可以说：内部诊断包显示这个方向有信号。HSA 三十六行中，模型直出 `16/36`，硬准入 `34/36`，支撑型窄补全后 `36/36`，多余最终卡片 `42`，低于全范围控制的 `195`。

可以说：公开压力表 PG40 当前还没有方法优势。强透明基线 `utility_density_greedy` 是 `25/40`，source-ledger 14B compiled 是 `11/40`，direct routing 五行是 `0/5`、utility `0.0987`，我们当前最好五行模型链路只有 `1/5`，utility `0.8155`，还不能支撑主方法结果。

## 对外不能说什么

不能说方法已经在公开基准上有效。

不能把 HSA-v0 三十六行当作公开基准胜利。

不能用支撑型窄补全的 `36/36` 直接写成主结果。

不能把 PG40 五行小样本包装成完整基准结论。

不能回避强基线：透明贪心、source-ledger、模型直出、结构化直出和理想上界必须同表出现。

## 当前证据分层

| 层级 | 证据 | 目前用途 |
| --- | --- | --- |
| 公开基准主线 | PerspectiveGap / PG40 | 对外方法有效性的主证据来源 |
| 公开切片压力 | PG40 tight-budget | 暴露预算、排序和来源角色问题 |
| 内部机制诊断 | HSA-v0 | 解释来源、范围、验证、证据不足和支撑补全 |
| 方法形状背景 | State Admission V1.1 | 说明 priority + executor 有方法形状，但任务仍偏合成 |
| 附表材料 | PACT、TypeCast、MATH、MAD/DAR/MOC | 解释失败面、解析风险和历史探索 |

## 对外主表应该长什么样

第一张主表应优先使用公开基准或公开切片。最小可报告版本已经落在 `docs/pg40_public_benchmark_main_table.md`，它被明确定义成公开切片压力表。

| 行 | 作用 | 当前状态 |
| --- | --- | --- |
| 直接路由 | 同模型直接基线 | 同五行 `0/5`，utility `0.0987` |
| 源账本 | 机制前身 | 同五行 `2/5`，utility `0.8498`；四十行 `11/40`，utility `0.8707` |
| 透明贪心 | 强规则基线 | 同五行 `5/5`，utility `1.0000`；四十行 `25/40`，utility `0.9825` |
| 结构化未编译 | 去执行器消融 | 同五行 `0/5`，utility `0.1803`，预算通过 `0.0000` |
| 我方编译后候选 | 主方法候选 | 同五行 `1/5`，utility `0.8155`，需要改进后再扩 |
| 上界 | 评分器上界 | 同五行 `5/5`；四十行 `40/40` |

HSA 可以作为第二张机制表。它解释为什么方法要有来源、范围、验证状态、证据不足和补全组件，但它不承担公开方法有效性结论。

## 下一步实验顺序

第一，停止把 HSA 扩包作为对外报告主线。HSA 后续只服务机制解释和错误定位。

第二，回到 PG40 / PerspectiveGap 公开切片。PG40 五行直接路由已补，source-ledger、structured no compiler、compiled、transparent greedy、oracle 已经保持同表；接下来补排序机制行。

第三，若当前 SSEAC 在 PG40 上仍低于 source-ledger 或透明贪心，先修 proposal 排序和预算感知单卡重排，暂缓写方法优势。

第四，若 PG40 小切片出现可解释增益，再扩到更大的公开切片或 PerspectiveGap 官方网格。

第五，对外报告只采用公开基准主表和机制附表组合：公开基准证明有效性，HSA 解释机制边界。

## 当前对外摘要草稿

我们提出一个测试时证据准入框架，用于多智能体通信中的来源、范围、验证和预算控制。当前结果显示，确定性准入执行可以在内部诊断包中显著减少证据越界和证据不足时的错误承诺；在公开切片 PG40 上，现有模型提案仍低于强透明基线，说明主要瓶颈在候选证据排序和预算感知选择。下一阶段将以公开基准同模型对照为主，报告直接路由、源账本、结构化未编译、我方编译后候选、透明贪心和上界的完整表格。

## 决策

项目主线从现在起改为公开基准优先。HSA 保留，但降级为机制诊断和附表。对外报告不再用 HSA 作为方法有效性的主证据。
