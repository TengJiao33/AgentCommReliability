## 当前成熟度判断

**结论：现在还不能说“我们的机制已经有效”，但也不能说这个想法不对。**

更准确的判断是：我们已经看到同伴中间状态里有可用搜索信号，但目前的传输和融合方式太粗，正信号和污染信号一起进来，导致收益被锚定、错位和过强干预抵消。

现阶段比较像一个 `solid_candidate`，不是完整故事。想法可以成立的最小前提是：

```text
自然生成过程中的中间状态确实携带搜索方向；
接收方必须能选择性地使用它；
通信不能让接收方续写发送方，也不能显式要求发送方写计划。
```

已有实验还没有证明第三点，所以不能急着扩大。

## 文献给我们的压力

### 一、选择性键值共享

相关文件：

```text
papers/text/kvcomm-selective-kv-sharing-2510.03346.txt
```

这篇最重要的启发不是“缓存一定有用”，而是“缓存要选择性共享”。它的核心做法是选少数关键层，把发送方键值对拼到接收方注意力可见的键值集合里，让接收方自己通过注意力决定看多少。

这对我们当前实现的压力很大：

```text
我们之前的 Pre-KV 更像把发送方前缀塞给接收方续写；
KVComm 更像给接收方额外可注意的材料；
这两者不是同一个干预强度。
```

所以，用户担心“不使用缓存是否错过 KVComm 的有效性”是对的。不能因为我们现有 Pre-KV 收益不稳，就否定 KV 缓存通道。更可能是我们使用缓存的方式不对。

### 二、状态变化轨迹

相关文件：

```text
papers/text/state-delta-trajectory-2506.19209.txt
```

这篇给了一个很关键的区别：传绝对隐藏状态不如传相邻生成步之间的状态变化。也就是说，真正有启发性的可能不是“发送方现在在哪里”，而是“发送方刚刚往哪个方向动了”。

这正好贴近我们想要的“无意识中间搜索状态”：

```text
绝对状态容易携带发送方身份、提示模板、局部错误、未完成答案；
状态变化更像搜索方向，干预更轻；
少数层注入比全层注入稳定。
```

文中对 Qwen2.5-7B 的层选择尤其值得参考：第 22 层是首选，前五候选还包括 24、9、20、12。我们之前用第 16 层，依据不够强。

### 三、跨上下文键值复用

相关文件：

```text
papers/text/kvcomm-cross-context-2510.12872.txt
```

这篇主要为了加速，不是为了推理提升，但它指出一个工程地雷：不同前缀、不同位置下的缓存不能天真复用，位置编码造成的错位可能比语义差异还大。

这解释了我们现有方案的一部分不稳定：

```text
发送方先生成一段；
接收方 prompt 接在发送方 past 后面；
接收方的位置、注意力历史、语境身份全被改了；
即使缓存里有搜索信号，也可能以错位形式进入。
```

下一版如果继续用 KV，应该尽量让多个智能体同步生成到相同位置再交换，或者至少把同伴 KV 作为额外旁路材料拼接，而不是改变接收方自身前缀位置。

### 四、潜通信综述

相关文件：

```text
papers/text/beyond-tokens-latent-communication-2606.05711.txt
```

这篇帮我们把设计空间拆成三问：

```text
传什么：键值缓存、隐藏状态、状态变化、软嵌入；
对齐到哪里：同层、跨层、同模型、异模型；
怎么融合：拼接、前置、加法、交叉注意力、恢复缓存。
```

我们的下一版应该先做最保守的格点：

```text
同模型；
同层；
少数层；
训练免；
低强度融合；
同题同伴状态与乱序/错题/随机状态强对照。
```

先不要引入学习型适配器，否则很难区分是通信机制有效，还是训练学会了一个新任务。

### 五、连续潜在推理

相关文件：

```text
papers/text/coconut-continuous-latent-reasoning-2412.06769.txt
```

这篇提醒我们：让模型在连续潜空间里稳定推理通常需要训练。推理时直接交换潜状态，模型未必知道怎么读。

这意味着我们的训练免版本不应该期待“大幅稳定提升”。合理目标应该更小：

```text
真实同题同伴状态显著优于随机状态；
救回率高于伤害率；
答案改变集中在原本分歧或低置信题上；
不会大规模破坏原本正确题。
```

### 六、嵌入辩论

相关文件：

```text
papers/text/ciphers-embedding-debate-2310.06272.txt
```

这篇的核心不是 KV，而是软信念：不要只传采样出来的一个词，而是传模型对词表分布的期望嵌入。

对我们的启发是，“无意识中间状态”可以不是文本计划，也可以是某一步的软分布、分布变化或高温分支信念。它保留了不确定性，和“不要主动生成完整计划”的要求更一致。

## 回答几个关键问题

### 真的有搜索信号吗？

我认为有，但现在证据级别是“局部可见”，不是“总体成立”。

支持证据：

```text
early-plan Pre-KV 小样本里有 BaW_to_C：相位平移、共轭根式、函数方程等题被救回；
这些救回不像纯随机扰动，因为它们方向上对应了发送方中间搜索路线；
KVComm 和状态变化文献都支持中间内部状态比离散文本更富含搜索信息。
```

反证或风险：

```text
同一批样本里 BaC_to_W 也不少；
安全潜状态变体基本只有 +1/-1；
同口径 question-only KV 只有很小净增益；
目前没有证明真实同题状态显著优于乱序同题、错题状态或随机状态。
```

所以现在不能说“信号强”，只能说“可能有，且被粗糙融合污染了”。

### 信号是怎么传给其他智能体的？

目前几条线其实传法不同：

```text
文字线索：通过自然语言提示传，损失大，容易自证错误；
Pre-KV：通过发送方 past_key_values 改变接收方后续生成的注意力历史，干预很强；
潜状态安全变体：通过隐藏状态残差或分支状态均值轻微 steering，干预较弱；
混合微承诺：文本短片段加缓存，但短片段截断会变成半截提示。
```

我现在认为最危险的是 Pre-KV 的“接续感”：接收方不是在看同伴的中间材料，而像是被放进了发送方的生成轨迹之后。这样容易把发送方未完成的局部错误当成自己的历史。

### 效果如何？

粗略排序：

```text
标准讨论：MATH500 从 364/500 到 378/500，稳定有用；
候选池守门：364/500 到 368/500，只小幅改善；
文字线索：364/500 到 357/500，伤害更大；
question-only KV 同口径：347/500 到 349/500，几乎没有实质收益；
early-plan Pre-KV 小样本：有救回也有伤害，净效应不稳；
潜状态安全变体：小样本 +1/-1，暂时不够。
```

这说明“讨论后的显式答案比较”目前比“潜通道”更稳。但潜通道还没有被以足够正确的方式实现。

### 错在哪？

我现在会把错误分成五类：

```text
一，显式计划污染：让发送方写计划会主动改变发送方，不符合原始想法；
二，前缀接续污染：接收方续写发送方 past，而不是选择性参考同伴状态；
三，位置错位污染：不同前缀长度和位置编码让 KV 语义不干净；
四，层选择粗糙：第 16 层、前 64 标记等选择依据不够，且你以前实验里已经提醒过前 64 标记可能有教训；
五，对照不足：没有把真实同题状态和错题状态、乱序状态、随机状态拆开，所以还不能证明是“搜索信号”而不是扰动。
```

## 下一版设计原则

必须保留用户提出的纯粹性：

```text
发送方不被要求写计划；
发送方只正常解题；
中间状态来自自然生成过程；
传递的是无意识副产物；
接收方不能被迫接受，只能低强度可选择地使用。
```

因此下一版不应叫“计划通信”，应叫：

```text
自然搜索状态交换
```

或者更窄一点：

```text
同步生成中的低强度潜状态旁路
```

## 推荐工程方案

### 第一阶段：只做诊断，不追求立刻提分

目标不是先刷 MATH500，而是证明真实同题中间状态是否有别于随机扰动。

数据：

```text
mca_disagreement_v1
mca_gold_contrast_v1
每个先跑 50 条，必要时扩到 100 条
```

条件：

```text
无通信基线；
真实同题同伴状态；
同题同伴状态打乱发送者；
其他题状态；
随机高斯状态；
零状态；
绝对隐藏状态；
状态变化量。
```

最低成立标准：

```text
真实同题状态的 BaW_to_C 明显高于其他题和随机状态；
真实同题状态的 BaC_to_W 不明显高于无通信；
真实同题状态在答案改变样本里的净收益为正。
```

如果做不到，这条 idea 至少在当前训练免设置下不成立。

### 第二阶段：同步截点，而不是前 64 标记

不要固定“前 64 标记”。建议改成同步生成截点：

```text
每个智能体正常生成；
在第 16、32、64、96 个解码步捕获状态；
也可以捕获状态变化范数最大的若干步；
接收方只在相同生成步附近接收同伴状态。
```

这样更接近“大家同时在想，偶尔泄露中间搜索状态”，而不是“一个人先写计划，另一个人接着写”。

### 第三阶段：优先实现状态变化量

最小实现：

```text
保存每个 agent 在选定层的 last-token hidden state；
计算 delta_t = h_t - h_{t-1}；
对同伴 delta 做均值或按范数裁剪后均值；
在接收方同层同位置加一个小尺度 residual；
scale 从 0.01、0.03、0.05 扫描；
只试层 22，再试 22+24，再试 22+24+9。
```

这个版本比 KV 拼接更容易实现，也更接近“方向启发”。

关键保护：

```text
范数裁剪；
只少数层；
只中间若干步；
不作用于最终答案解码的全部过程；
不把同伴状态写入 past_key_values；
保存每题每步的 delta 范数和答案转移。
```

### 第四阶段：再实现选择性 KV 旁路

如果状态变化量显示真实同题优于随机，再做 KV 版。

KV 版不建议继续 Pre-KV 接续，而建议：

```text
接收方保留自己的正常 past；
在选定层，把同伴同位置附近的 K/V 作为额外 memory 拼到 attention 可见集合；
同伴 K/V 只来自少数层；
可加门控系数或 top-k attention 过滤；
不改变接收方自己的 position_ids。
```

这比现在工程量大，因为要改模型 forward 的注意力逻辑，但概念上更对。

### 第五阶段：嵌入或软分布分支

如果用户想保留更强“随机、无意识、非计划”的性质，可以另做软分布版本：

```text
发送方正常解题；
在若干步记录 logits 分布或 top-k logits；
转成期望嵌入或分布变化；
接收方只把它作为弱 residual 或额外软 token；
对照高温分支、低温分支、随机分布。
```

这条线的好处是工程上比 KV 轻，且不直接传可读计划。

## 最小下一步

我建议下一步只做一个很小但杀伤力强的实验：

```text
名称：natural-search-delta-v0
模型：Qwen2.5-7B-Instruct
层：22
截点：第 16、32、64、96 解码步
通信对象：同题同伴 delta 均值
对照：无通信、真实同题、其他题、随机高斯、绝对状态
样本：mca_disagreement_v1 前 50 条
指标：BaW_to_C、BaC_to_W、答案改变率、真实同题相对随机的净收益
```

这一步的价值不是分数，而是回答：

```text
真实自然中间状态到底是不是搜索信号？
```

如果真实同题状态不优于其他题和随机状态，就该暂停潜通道，回到显式答案比较或候选池守门。如果真实同题状态明显更好，再投入更重的选择性 KV 旁路实现才值得。
