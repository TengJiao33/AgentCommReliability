# State Admission V2 Abstention Gate Ablation

日期：2026-06-18

## 核心判断

显式最终裁决闸门没有修复强制作答。模型会填写闸门字段，但它常常把不可见、未核验或已拒绝事实继续算入最终裁决者的证据集合。

这轮结果让主问题更窄：当前最值得压的是“模型能否正确重构最终裁决者实际可见且已核验的事实集合”。

## 证据链

本地门检显示，`abstention_explicit` 提示在 9 行上全部可物化，平均约 7064 字符，没有泄漏答案或评分字段。人工基线在同一评分器下仍是全通过。

GPU run 是 `experiments/20260618-1848-a8002-state-admission-v2-abstention-explicit-smoke9-qwen25-14b/`。模型为 Qwen2.5-14B，单卡，温度为零，9 行全部完成。

结果是 `strict=0.1111`、`downstream_ok=0.4444`、`option_state_recall=0.8148`。上一轮同模型 `option_state_first` 是 `strict=0.1111`、`downstream_ok=0.4444`、`option_state_recall=0.9259`。显式闸门没有提高最终裁决正确性，还降低了候选状态恢复。

## 机制解释

成功例很少但有用。`hb10_b_hazard_quarantined` 中，模型把 Warehouse B 标成证据不足，闸门未通过，最终回答为空。这说明新格式在某些核验拒绝场景下能工作。

主要失败发生在证据集合重构。`hb10_c_enabler_no_final_scope` 中，模型知道 Warehouse C 的支撑事实只准入给规划角色，却仍在最终闸门里把 Warehouse C 算成可行。`hb11_school_repair_quarantined` 中，模型一边拒绝修复事实，一边把同一事实用于 School Gym 可行判断。

这说明当前失败不是缺少“是否唯一可行”的步骤。模型执行这个步骤时，仍然把来源、核验状态和可见范围压扁成普通任务证据。

## 对文章故事的影响

这轮结果支持把文章对象继续收窄。更可信的说法是：在多角色决策中，模型难以维持“最终裁决者实际可用证据集合”和“最终答案”之间的一致性。

这也给方法部分一个更具体的方向。下一版不应继续堆自然语言规则，而应把最终裁决者可见且已核验的事实集合显式物化，要求模型只在这个集合上生成候选状态和最终答案。

## 失败和边界

样本仍然只有 9 行，不能承载规模化结论。

提示更长，可能带来额外负担。候选状态恢复下降，说明更强约束也可能损伤前面的语义判断。

评分器不直接评分闸门字段。这轮关于闸门自相矛盾的判断来自人工逐例检查，下一轮应把闸门一致性变成正式指标。

## 下一步压力测试

下一步应做 `visible_facts_first`。提示先要求模型输出最终裁决者可见且已核验的事实编号集合，再要求候选状态和最终答案只能引用该集合。

成功信号是扰动行的证据不足判断上升，同时候选状态恢复不明显下降。失败信号是可见事实集合已经正确，但模型仍然强行给答案。后者会更直接支持“可准入状态到最终裁决的一致性失败”。
