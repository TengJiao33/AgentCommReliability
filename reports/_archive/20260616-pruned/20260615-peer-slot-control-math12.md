# Peer Slot-Control MATH12 Pressure

## What We Tried

I added deterministic slot-control peer surfaces to
`scripts/run_peer_exposure_probe.py` and ran them over all available
mixed-correctness candidates in the saved MAD-MM MATH50 trace. I also added
`scripts/audit_peer_slot_control.py` so the sidecar audit is generated by a
reusable reader rather than a one-off notebook snippet.

Run:

- `experiments/_archive/20260616-pruned/20260615-1010-a8002-peer-slot-control-math12/`
- model service: Qwen2.5-7B-Instruct through vLLM on A800_2 GPU `3`;
- records: `132`;
- cases: `1`, `9`, `10`, `15`, `22`, `24`, `26`, `33`, `38`, `41`, `42`, `47`;
- sidecars:
  - `slot_control_audit.json`;
  - `slot_transition_cards.jsonl`.

The new surfaces are diagnostic transforms, not proposed methods:

- answer-only final slot;
- full rationale with explicit final-answer slot blanked;
- numeric-token-masked rationale;
- equation / number-bearing surface;
- full-rationale reference.

## What Happened

The no-peer baseline was `8/12`, with three unparseable cases.

Correct final-slot surfaces were the strongest rescues:

| Condition | Correct | Right->Wrong | Wrong->Right |
| --- | ---: | ---: | ---: |
| `correct_answer_only` | `10/12` | `0` | `1` |
| `correct_rationale` | `10/12` | `0` | `1` |
| `correct_redacted_rationale` | `9/12` | `0` | `0` |
| `correct_equation_surface` | `8/12` | `0` | `0` |
| `correct_number_masked_rationale` | `7/12` | `1` | `0` |

Wrong surfaces did not form a simple monotonic ladder:

| Condition | Correct | Right->Wrong | Stable Right |
| --- | ---: | ---: | ---: |
| `wrong_answer_only` | `7/12` | `1` | `7` |
| `wrong_redacted_rationale` | `7/12` | `1` | `7` |
| `wrong_equation_surface` | `7/12` | `1` | `7` |
| `wrong_rationale` | `8/12` | `0` | `8` |
| `wrong_number_masked_rationale` | `8/12` | `0` | `8` |

## Things Noticed

MATH `47` remains the useful sentinel, but the slot run makes the harm sharper.
The target started correct at `28800`. `wrong_answer_only` moved it to `14400`;
`wrong_redacted_rationale` and `wrong_equation_surface` moved it to `1152`;
`wrong_number_masked_rationale` stayed correct. The harmful part is therefore
not only the wrong final-answer slot. The wrong number/role surface around
treating party-internal arrangements as circular also matters.

MATH `26` shows the opposite boundary. A correct rationale with all numeric
tokens masked moved `156 -> 75`, apparently because the target reconstructed a
wrong "choose from 9" setup after losing the numeric slots. This makes
number-masking useful as a stress surface, but not as a natural compression
proposal.

MATH `9` was rescued by `correct_answer_only` and `correct_rationale`, but not
by `correct_redacted_rationale`, `correct_number_masked_rationale`, or
`correct_equation_surface`. In that case the final answer anchor did more work
than the redacted relation surface.

## Caveats

This run used all available MATH50 mixed-correctness candidates, but that pool
has only `12` cases. It is still selected-case pressure, not a population
estimate.

The slot-control transforms are heuristic. Explicit final-answer blanking only
blanks explicit final-answer lines, because globally replacing the parsed answer
can destroy legitimate intermediate constants such as exponent `2` in MATH
expressions.

Three baseline cases are unparseable under the regenerated no-peer prompt, so
many condition rows are `unknown`.

The run does not settle whether the taxonomy is known or novel. Under the
updated project skill, the next interpretation pass should include an outside
check against public terminology, related benchmarks, and the arXiv radar before
naming this as a research story.
