## Things Noticed

The real traces now share the same top-level vocabulary as the toy communication-regime harness. That matters because the harness is no longer a separate sketch: its fields can be used to label existing reproduced systems.

The negative fact changed shape after this re-extraction. The real traces now have derived recipient-context events, but not raw prompt-level context states. MAD-MM context events come from `mask_history`; DAR context events come from `retention_events`; MOC context events come from `ism_result` sidecars.

This sharpens the next contact point:

```text
public state is now represented;
recipient context is partially reconstructed;
prompt-level/generated context is still missing in real traces.
```

## Caveats

- The DAR and MOC labels are manual and should be used sparingly. MAD-MM MATH50 public-state labels are generated by method-specific extractor defaults.
- `context_events` in these real traces are derived from masks, retention events, or ISM sidecars. They are not raw prompt logs.
- `saturated_arithmetic` is a coarse label for GSM8K-style slices here; it means these slices have tended to hide communication necessity, not that every item is trivial.
- MOC transition type remains `unknown` because the extractor has no first-round independent baseline for those detail files.
- This is a schema/contact step, not a new model result.
